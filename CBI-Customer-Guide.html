Get Started with Browser Isolation

Step-by-step instructions to configure and deploy Cloud Browser Isolation in your environment.

Step 1

Create an Isolation Profile

Administration > Cloud Browser Isolation > Isolation Profiles

The isolation profile controls the user experience during isolated sessions ‚Äî what they can and can't do.

  • Click Add Isolation Profile
  • Name it (e.g., "Standard - Restrict Downloads")
  • Clipboard: Block copy/paste to prevent data exfiltration (or Allow for low-risk use cases)
  • Downloads: Set to CDR (Content Disarm & Reconstruction) to flatten files into safe PDFs, or Block entirely
  • Upload: Block or Allow depending on your data protection requirements
  • Printing: Block to prevent data leakage via print-to-PDF
  • Rendering: Select Turbo Mode for near-native 50fps experience
  • Persistence: Choose non-persistent (session ends when tab closes) for maximum security
Full profile configuration guide ↗
Step 2

Create a URL Filtering Rule with Isolate Action

Policy > URL & Cloud App Control > URL Filtering Rules

This is where you tell Zscaler which traffic to isolate instead of allowing or blocking.

  • Click Add URL Filtering Rule
  • Name: "Isolate Uncategorized Sites" (start here ‚Äî highest risk category)
  • URL Categories: Select "Uncategorized" and optionally "Newly Registered Domains"
  • Action: Select Isolate
  • Isolation Profile: Select the profile you created in Step 1
  • Users/Groups: Start with a pilot group (e.g., IT or Security team) before rolling out broadly
  • Rule Order: Place above any Allow rules for the same categories
Full URL filtering policy guide ↗
Step 3

Enable Smart Browser Isolation (if licensed)

Policy > Cloud Browser Isolation > Smart Isolation

Smart Isolation uses AI to automatically isolate suspicious pages ‚Äî even on categorized, "trusted" sites that have been compromised.

  • Enable Smart Browser Isolation
  • Set the risk threshold ‚Äî start with the default and adjust based on volume
  • Assign to user groups ‚Äî can run alongside your URL-based isolation rules
  • This catches threats that URL category-based isolation misses (e.g., compromised news sites)
Smart Isolation setup guide ↗
Step 4

Test and Validate

Before rolling out broadly, verify isolation is working correctly with your pilot group.

  • As a pilot user, visit an uncategorized site ‚Äî you should see the isolation indicator in the browser
  • Test clipboard, download, and print controls match your profile settings
  • Verify Turbo Mode rendering is smooth (should feel near-native)
  • Check Analytics > Web Insights for isolation activity logs
  • Gather pilot user feedback ‚Äî are there usability issues? Sites that break?
  • Once validated, expand the user scope in your URL filtering rule
Step 5

Expand Coverage

After successful pilot, expand isolation to more categories and user groups.

  • Add more URL categories: "Newly Registered Domains," "Other Miscellaneous," "Web-Based Email"
  • Create separate rules with different profiles (e.g., stricter controls for contractors vs. employees)
  • Consider isolating specific cloud apps for BYOD users (SaaS apps via ZPA isolation)
  • Review isolation analytics monthly ‚Äî adjust categories based on threat trends
